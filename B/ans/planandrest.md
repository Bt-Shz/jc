## My prewriting summary

Main ideas from prewriting:

### Three changes/additions to ACM Code

1. Lifecycle accountability / impact audits
- The Code should require risk review before deployment and monitoring after deployment.
- “Avoid harm” should mean documented risk assessment, fallback plans, user feedback channels, and post-launch correction.
- Especially important for AI/ML systems with uncertainty, unknown-unknown failures, and false high confidence.

2. Measurable fairness and bias testing
- The Code already says “be fair and do not discriminate,” but this is too general.
- It should require professionals to test for bias and explain what fairness definition they used.
- Bias can appear through imbalanced data, correlated variables, or badly framed problems.
- Removing race/gender is not enough if other features reproduce discrimination.

3. Privacy, user agency, and anti-manipulation
- The Code should strengthen privacy into meaningful data control.
- Users should know and control what is collected, inferred, remembered, retained, shared, or deleted.
- For LLMs and AI companions, user agency also means avoiding designs that exploit emotional dependence or fake intimacy.
- Consent is not meaningful if users cannot understand what the system does with their data.

### When/how technologists should first be introduced to the Code

- Introduce it in the first CS/programming course, before students treat coding as ethically neutral.
- Reintroduce it in AI, ML, data science, software engineering, capstone, internship, and workplace onboarding.
- Do not teach it as a one-time checkbox.
- Use case studies, risk audits, design reviews, and deployment decisions.
- Require agreement before students/professionals access real user data or deploy high-impact systems.

### Consequences for violation

- Consequences should be proportional.
- Honest mistake:
  - correction plan
  - ethics training
  - review
- Negligence or repeated violation:
  - formal discipline
  - removal from project
  - required supervision
- Intentional concealment, exploitation, falsified audits, or serious harm:
  - job termination
  - professional sanctions
  - legal referral where appropriate
- ACM membership termination alone is too weak because many technologists are not ACM members.
